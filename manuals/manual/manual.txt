Before you start:
1. After installing the Python virtual environment (see `installation.txt`), change directory to the project root (`./hotfilm_NN`) and use that folder for all commands.

2. Activate the virtual environment:
	 $ source hotfilm_env/bin/activate
	To deactivate, run:
	 $ deactivate

3. All commands assume the current working directory is `./hotfilm_NN`.

Use a consistent series identifier (e.g., `xxx`) for folders and files used by the scripts.

Training
 - Train a network -

Requirements:
 - Hot-film voltage CSV sampled at 2000 Hz.
 - Sonic anemometer velocity CSV sampled at 20 Hz.

1. Create a folder named `{serie}` under:
	data/raw

2. Place the two CSV files inside that folder:
	data/raw/{serie}/hotfilm_{serie}.csv
	data/raw/{serie}/sonic_{serie}.csv

3. Generate the training CSV by running:
	$ python3 create_csv.py train {serie}

4. Start training with:
	$ python3 train_mlp.py {serie}

Training outputs (models and metadata) are saved under `data/train/train_results/` and `models/`.

Run (Inference)
1. Place the voltage data CSV in:
	data/raw/{serie}/hotfilm_{serie}.csv
2. Prepare the run CSV:
	$ python3 create_csv.py run {serie}
3. Run inference:
	$ python3 run_mlp.py {serie} {model_filename}.pth

Output files are stored under `data/run/results/velocity_{SERIE}/`.

Spectrum (Power Spectral Density) and Dissipation (ε)
 - Use `spectrum.py` to plot PSDs for predicted and sonic signals and optionally compute the dissipation rate (ε).

To compute ε or run theoretical validation, provide a configuration JSON at `data/config/config_{SERIE}.json` containing kinematic viscosity, sampling rates, and `EPSILON_EXPECTED`. Optionally include a theoretical spectrum file for validation.

Run spectrum analysis:
	$ python3 spectrum.py {SERIE}

The script will ask whether to compute ε from the predicted model and whether to run theoretical validation; these operations require the config JSON.
